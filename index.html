Правила коммунального сервера

⚠️ Важно: Сервер общий — нарушение правил ломает работу других команд!
  • 01 Работай только в своей папке — /home/ВАШ_ЛОГИН/
  • 02 Не трогай чужие контейнеры — docker stop/rm/restart только своих
  • 03 Используй только свой порт из таблицы ниже
  • 04 Называй контейнеры и тома с префиксом своей команды
  • 05 Не меняй системные настройки без согласования со старшим админом
  • 06 Не забываем делать бэкап БД

Развёртывание

1
Создать папку проекта (в текушей личной директории)
bash
mkdir ~/odoo && cd ~/odoo
2
Создать docker-compose.yml
bash
// Либо другой редактор, либо с помощью scp скинуть yml с хоста на сервер.
// Пример с scp: scp docker-compose.yml username@204.12.253.210:/home/username
nano docker-compose.yml

Вставить шаблон ниже, заменив НОМЕР на номер команды и ПОРТ на свой порт из таблицы:

yaml — шаблон
services:

  db:
    image: postgres:16
    container_name: teamНОМЕР-db
    environment:
      POSTGRES_USER: odoo
      POSTGRES_PASSWORD: odoo
      POSTGRES_DB: postgres
    volumes:
      - teamНОМЕР_db_data:/var/lib/postgresql/data
    restart: unless-stopped

  odoo:
    image: odoo:18
    container_name: teamНОМЕР-odoo
    depends_on:
      - db
    ports:
      - "ПОРТ:8069 (Поставить порт вашей команды!)"
    environment:
      HOST: db
      USER: odoo
      PASSWORD: odoo
    volumes:
      - teamНОМЕР_odoo_data:/var/lib/odoo
    restart: unless-stopped

volumes:

Сохранить файл: Ctrl+O → Enter → Ctrl+X

3
Запустить контейнеры
bash
docker compose up -d

Первый запуск скачает образы — займёт 2–5 минут. Проверить статус:

bash
docker compose ps
Оба контейнера должны показывать статус Up. Если один из них Exit — смотри логи: docker compose logs odoo

Первый вход в Odoo

Открой в браузере свой адрес из таблицы портов. Появится страница создания базы данных.

После нажатия Create Database подожди 3–5 минут — Odoo инициализирует базу данных.

docker compose down

docker compose exec odoo bash
Зайти внутрь контейнера Odoo
docker compose exec db bash
Зайти внутрь контейнера базы данных

Бэкап и восстановление

⚠️ Не забываем бэкап БД!

Создать бэкап

bash
docker compose exec db pg_dump -U odoo odoo > backup_$(date +%Y%m%d).sql

Файл сохранится с датой в названии — например backup_20260315.sql

Восстановить из бэкапа

bash
docker compose exec -i db psql -U odoo odoo < backup_20260315.sql

Что делать если что-то сломалось

🔴 Контейнер не запускается
Смотри логи — там будет причина:
bash
docker compose logs odoo
🔴 Хочу начать с чистого листа
⚠️ Удалит контейнеры И все данные!
bash
docker compose down -v
docker compose up -d
🔴 Контейнер завис
bash
docker compose restart odoo
🔴 Забыл пароль от Odoo
Зайди в Settings → Users → поменяй пароль через интерфейс.
